Conjoon vs. RoundCube Comparison
Licensing for Commercial Use
RoundCube:

Licensed under the GNU General Public License (GPL)
Free to use commercially, but requires that any modifications you make must also be distributed under GPL
You can deploy it within your business without additional licensing costs
If you modify the code and distribute it, you must make your modifications open source

Conjoon:

Originally released under GPL v3 license
Later versions offer dual-licensing: GPL and commercial licensing options
Commercial license allows you to use and modify the software without sharing your changes
Typically requires payment for commercial deployments depending on the specific version

Other Key Differences
User Interface:

RoundCube: Clean, traditional webmail interface with a focus on functionality
Conjoon: More modern UI with more advanced layout options and customization

Features:

RoundCube: Solid core email functionality, widely used and tested
Conjoon: More focus on groupware features beyond email (calendaring, task management)

Development Activity:

RoundCube: Larger community, more active development, and more frequent updates
Conjoon: Smaller developer base, potentially slower update cycles

Integration:

RoundCube: Extensive plugin ecosystem and better third-party integration
Conjoon: Better integrated suite of tools, but fewer external integration options

Deployment:

RoundCube: Easier to deploy, more documentation, and community support
Conjoon: May require more technical expertise to deploy and maintain

Performance:

RoundCube: Generally better performance with large mailboxes
Conjoon: May handle complex groupware tasks better but sometimes at the expense of email performance

Commercial Use Recommendation
If you're planning to use either for commercial purposes:

RoundCube is more straightforward from a licensing perspective if you don't plan to modify the code extensively or can comply with GPL requirements
Conjoon with a commercial license would be better if you need to make proprietary modifications without sharing your code
